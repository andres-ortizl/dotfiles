specdex — list view cell redesigns · round 2

Iterating two cells of the fleet List view: agents (now "show only who's working") and review (more zen). Phase and PR are locked to their selected designs below for reference. Every variant sits in a real .list-row against its neighbors, in both light and dark. Palette discipline holds: blue is the only interactive accent, green only means an active agent, ember only means a human is needed, health lives in the dot alone.

1 · Agents — show only who's working

New instinct: stop rendering idle/absent agents. The cell shows only active:true agents and sits quietly when nobody is driving — the health dot already carries idle vs working, so the agents cell needn't repeat it. Watch parse-cache (coder active), timeseries-perf (reviewer active), fleet-watch (idle, nobody driving → empty), verify-flake (coder active), and typed-create-proxy (done, no agents → empty). Tradeoff noted per variant: only-active drops the "a reviewer is assigned but idle" signal — in a dense ambient list that's the right call.

2 · Phase — locked

Selected last round. The 8-node rail + separate label collapse into one ~112px cell: a per-phase glyph + the label. Current phase glyph is accent blue; a done/accepted spec is fully neutral (no blue cursor).

3 · PR — locked

Selected last round. A small PR/branch glyph leads, its color carrying lifecycle (blue open · neutral merged · ember closed); the number trails in mono, tabular. Empty state is a quiet em-dash.

4 · Review — quieter, cooler

Round 1 felt busy. Review meta is secondary — it should whisper, never compete with the name or phase. These are restrained: pip meters, demoted rounds, a one-glance score. Right-aligned, tabular. Rows shown: 0-review (vanishes), r1 (no score), r2 ★4, r1 ★5.
